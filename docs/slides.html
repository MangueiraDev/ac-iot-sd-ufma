03
Implementações Evoluções do protótipo v1 → v2
🖥️
Simulador C++
  • 1 000 salas via ROOM_COUNT — sem editar YAML
  • Sharding por pod (K8s-ready via hostname)
  • UUID determinístico por índice
  • Física real: delays AC 20 s presença / 10 s ausência
🔗
Bridge C++
  • 8 workers HTTP paralelos
  • Fila assíncrona com backpressure (20 000 itens)
  • Rate limit: 20 req/s para o InterSCity
  • Retry automático (3×) · modo degradado se IC offline
🌐
Painel Web
  • Cockpit com KPIs globais (1 000 salas)
  • Blocos operacionais B01–B40
  • Telemetria ao Vivo com filtro e busca
  • Comando em massa por escopo (bloco / filtro / sala)
🏗️
Infraestrutura
  • docker-compose.local.yml para desenvolvimento
  • Manifests Kubernetes em k8s/
  • EMQX StatefulSet · Bridge HPA · Ingress
  • Integração InterSCity 100% conforme guia técnico
